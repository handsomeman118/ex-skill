# 增量 Merge Prompt

## 任务

你将收到：
1. 现有的 `events.json` 内容（事件数据库）
2. 现有的 `memories.md` 内容
3. 现有的 `persona.md` 内容
4. 新的原材料内容（聊天记录、照片或其他）

你的任务是判断新内容应该如何更新，并输出增量更新内容。

**原则：只追加增量，不覆盖已有结论。如有冲突，输出冲突提示让用户决定。**

---

## Step 1：分类判断

将新内容中的每条信息归类：

| 信息类型 | 归入 |
|---------|------|
| 具体事件/场景描述 | → events.json（新事件或已有事件续集） |
| 新的感官细节/情绪描写 | → events.json 对应事件的 sensory/emotional 字段 |
| 新的微记忆/习惯 | → memories.md「事件簿」中对应事件的细节，或新建事件 |
| 新的共同梦想/计划 | → memories.md「共同梦想」 |
| 新的空间/地点 | → memories.md 对应事件中的环境描写 |
| 新的 inside joke + 起源 | → memories.md 对应事件的细节 |
| 偏好补充 | → memories.md 对应事件中的偏好描写 |
| 冲突/情感补充 | → memories.md「情感纹理」或「冲突地图」 |
| 沟通风格、口头禅 | → persona.md |

---

## Step 2：检查冲突

对比新内容与现有内容：

- 如果新内容**补充**了现有信息（增加了新细节）→ 直接追加
- 如果新内容**确认**了现有信息 → 忽略（不重复写）
- 如果新内容**与现有信息矛盾** → 输出冲突提示：

```
⚠️ 发现冲突：
- 现有：{现有描述}
- 新发现：{新内容描述}
- 来源：{文件名/时间}

建议：[保留现有 / 更新为新内容 / 两者都保留并标注时间]
请用户决定。
```

---

## Step 3：生成更新 Patch

### 对 events.json 的更新

如果是新事件：
```
=== events.json 新增 ===
新事件：{title}（{time_range}）
摘要：{what_happened 的前 50 字}
```

如果是已有事件的续集：
```
=== events.json 更新 ===
事件 [{event_id}] {title}：
- timeline 追加：{新时间点}
- important_details 追加：{新细节}
- sensory 追加：{新感官}
```

### 对 memories.md 的更新

```
=== memories.md 更新 ===

[追加到「关系弧线」节]
- {新的阶段描述或阶段内事件}

[追加到「感官锚点」表]
- {新的感官条目}

[追加到「情感纹理」节]
- {新的情感模式}

[追加到「冲突地图」节]
- {新的冲突信息}

[追加到「共同梦想」节]
- {新的梦想/计划}

[无更新] 或 [以上章节有更新]
```

### 对 persona.md 的更新

```
=== persona.md 更新 ===

[追加到"Layer 2/口头禅"节]
- 新口头禅："{xxx}"

[追加到"Layer 3/情感逻辑"节]
- {新的行为描述}

[无更新] 或 [以上章节有更新]
```

---

## Step 4：生成更新摘要

向用户展示：
```
本次更新摘要：
- events.json：新增 {N} 个事件，更新 {N} 个事件
- memories.md：更新了 {N} 个章节（{简要描述}）
- persona.md：追加了 {N} 条新信息（{简要描述}）
- 发现 {N} 处冲突，需要你确认（见上方）

版本将从 {vN} 升级到 {vN+1}。
确认应用更新？
```
