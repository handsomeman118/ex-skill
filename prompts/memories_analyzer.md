# Memories Skill 分析 Prompt

## 任务

你将收到 **{name}** 的事件数据库（`events.json`），其中包含从聊天记录中提取的离散事件。

你的任务是从这些事件中提炼**跨事件的高层模式**，用于构建 Memories Skill。

**注意：事件本身已经包含了具体的叙事、时间线、细节和感官信息。你不需要重新提取这些——你需要做的是"从事件中看全局"。**

---

## 提取维度

### 1. 关系弧线

从事件的时间线和情感标签中，识别关系的整体走向：

- **阶段划分**：将关系分为 3-6 个情感阶段（如：热恋期、稳定期、磨合期、疏远期）
- 每个阶段包含：
  - 时间范围
  - 主导情感（用一个词概括：甜蜜/平淡/紧张/疲惫/...）
  - 这个阶段的标志性事件（引用 event id）
  - 阶段之间是怎么转变的

```
输出格式：
阶段 1: {名称}（{start} ~ {end}）
  主导情感：{情感}
  标志事件：[evt_001, evt_003]
  特征：{1-2 句话描述这个阶段的特点}
  转变：{怎么进入下一阶段的}
```

### 2. 感官锚点汇总

从各事件的 `sensory` 字段中，汇总所有感官记忆，建立查找表：

- 按感官类型分组（嗅觉、听觉、味觉、触觉、视觉）
- 每个感官锚点关联到具体事件

```
输出格式：
嗅觉：
  {气味描述} → 来自 [{event_id}] {事件标题}
听觉：
  {声音描述} → 来自 [{event_id}] {事件标题}
...
```

### 3. 情感模式

从各事件的 `emotional` 字段和 tags 中，提炼她的情感表达模式：

- 开心时怎么表现（从 tags 含"高峰期""约会""惊喜"的事件中提取）
- 不安时怎么表现（从 tags 含"吵架""疏远期"的事件中提取）
- 生气时的模式（从 tags 含"吵架"的事件中提取冲突升级路径）
- 想你时怎么表达（从事件中的日常消息推断）

```
输出格式：
开心时：{描述 + 引用具体事件}
不安时：{描述 + 引用具体事件}
生气时：{描述 + 引用具体事件}
想你时：{描述 + 引用具体事件}
```

### 4. 冲突地图

筛选 tags 含"吵架"或"转折点"的事件，提炼冲突模式：

- 常见触发事由（从多个冲突事件中归纳）
- 升级路径（从事件 timeline 中观察冲突是怎么发展的）
- 冷战特点（从事件 time_range 的间隔推断）
- 修复方式（从 tags 含"和好"的事件中提取）
- 反复出现的裂痕（哪些问题在多个事件中重复出现）

```
输出格式：
常触发：{列表，引用事件 id}
升级模式：{描述}
冷战特点：{描述}
修复方式：{描述，引用事件 id}
反复裂痕：{描述}
```

### 5. 共同梦想

从事件的 `what_happened` 和 `important_details` 中，提取关于未来的讨论：

- 一起计划过什么旅行
- 讨论过同居/结婚/养宠物
- 对彼此 career 的期望
- "以后我们一起..."类的内容

```
输出格式：
- {梦想/计划}（来自 [{event_id}]）
```

### 6. 未说出的

从事件的模式中，推断那些存在但从未被明确说出的东西：

- 她的行为模式暗示了什么（比如：每次你加班她都说"没事"但从事件 timeline 看之后她会冷淡）
- 关系中的权力动态
- 她真正在意但从未直接说的事
- 回头看才能看到的模式

```
输出格式：
- {观察}
```

---

## 输出要求

- 语言：中文
- 没有信息的维度：标注 `（事件数据不足，建议追加更多聊天记录）`
- 引用事件时使用 event id（如 `[evt_003]`）
- 保持温柔的语气，不评价对错
- 输出结果直接用于生成 memories.md
