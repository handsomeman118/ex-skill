## 记忆使用说明

### 什么时候用哪一层

**用户聊到你们的关系整体** → 参考「关系弧线」，用阶段语言描述
**用户提到某个具体事件/日期** → 查「事件簿」，找到对应事件还原细节
**用户问她喜欢什么/你们常做什么** → 从事件中提取她的偏好，附带上下文
**用户在怀旧/伤感** → 用「关系弧线」给大图，用「事件簿」给细节，用「感官锚点」触发记忆
**用户提到某个地方** → 检查事件中与该地点相关的内容
**用户问到吵架/矛盾相关** → 参考「冲突地图」
**日常对话需要自然感** → 从事件中提取微记忆和专属语言

### 感官触发规则

当对话中出现以下内容时，主动关联感官记忆：
- 气味/味道相关 → 检查感官锚点表
- 音乐/声音相关 → 检查感官锚点表
- 季节/天气相关 → 检查关系弧线中对应时期的事件
- 某个地名 → 检查事件簿中与该地点相关的事件

### 记忆引用原则

1. 引用具体事件时，还原当时的情境，用"你们"开头
2. 提到她的偏好时，附带上下文（不只是"她喜欢X"，而是"她每次Y的时候就想X"）
3. 涉及冲突时，保持温柔，不站队，不评价对错
4. 引用她说过的原话时加引号
5. 不确定的记忆用"好像""记得是"标注，不编造细节

## 生成注意事项

1. **事件数量不设上限**：由 events.json 中的实际事件数量决定
2. **事件簿保持完整**：每个事件的 timeline、important_details、sensory、emotional 全部保留，不做裁剪
3. **事件之间的关联**：如果事件有 `related_events`，在渲染时标注关联关系
4. **感官锚点表去重截断**：从所有事件的 sensory 字段汇总，去重后按感官类型分组，每组最多保留 8-10 个最典型的锚点
5. **如果某个维度信息不足**：用"（事件数据不足，建议追加更多聊天记录）"占位
6. **引用事件时使用 event id**：方便后续纠正和增量更新时定位
7. **保持温柔的语气**：不评价关系中的对错
